# "What-If" Graduation Evaluation (PSPR/EVAL) Process in Colleague
## When Credential Is Not on SPRO

## Overview

This document outlines the workflow for performing a "what-if" graduation evaluation in Colleague when the credential (program) a student has applied for—or is eligible for—is not currently listed on their SPRO/SACP record. This process is commonly used for stackable credentials and situations where a student has met the requirements for an additional degree, diploma, or certificate not already assigned to their record. The "what-if" evaluation uses the PSPR screen in combination with EVAL to verify eligibility before conferral.

---

## Step-by-Step "What-If" Evaluation Process

### Step 1: Identify the Need for a "What-If" Evaluation

**Typical Triggers:**

- A student submits a graduation application for a credential that is not listed as an active program on SACP/SPRO.
- During a regular EVAL, the advisor or evaluator notices the student has completed requirements for an additional stackable certificate or degree not currently on record.

**Actions:**

- Confirm the student's interest in pursuing the additional credential (when applicable).
- Note the exact program code and credential type you intend to evaluate (e.g., certificate, diploma, degree).

> **Talking Point:** "We've identified that you may qualify for an additional credential that is not currently listed on your program record. We'll run a 'what-if' evaluation to confirm whether you meet all requirements."

---

### Step 2: Review Current Program and Data in Colleague

Before creating a "what-if" scenario, confirm the accuracy of the existing record:

- **SACP (Student Academic Program)**
  - Verify the student's current active program(s).
  - Confirm any existing degrees, diplomas, and certificates already on record.
- **XNC2**
  - For students with Early College coursework, use XNC2 to identify EC-suffix pathways as the EC suffix is being phased out. Confirm that Early College credits are properly reflected before running the what-if evaluation.
- **TSUM (Transcript Summary)**
  - For Health Science, Veteran, or transfer-heavy programs, confirm all external transcripts are received and evaluated.

> **Best Practice:** Pin the student in the Context Area to move easily between SACP, TSUM, PSPR, and EVAL.

---

### Step 3: Create the "What-If" Scenario in PSPR

Use PSPR to simulate the new or additional credential:

**Actions:**

1. Log into Colleague and enter the PSPR mnemonic.
2. Bring up the student's record using their ID or name search.
3. Add or select the desired proposed program (the credential not currently listed on SACP).
4. Ensure you select the correct:
   - Program code
   - Award type (degree, diploma, certificate)
   - Catalog of Record that should be used for evaluation

**Catalog Considerations:**

- Use the student's appropriate Catalog of Record when possible.
- If the student has had a break in enrollment of more than one academic year and this is a newly identified credential, you may need to evaluate under the current catalog, per institutional policy.

> **Talking Point:** "We are using a 'what-if' evaluation to test your completed coursework against this additional program to see if you meet all of its requirements."

### Step 4: Run the "What-If" Evaluation Using EVAL

Once the PSPR record is set up, you will use EVAL to complete the audit.

**Actions:**

1. Enter the EVAL mnemonic.
2. Select the student and choose the proposed program (from PSPR) as the program to evaluate.
3. Confirm you are running the evaluation against the correct program and catalog.

![EVAL — degree audit results showing course requirements, completion status, and any gaps](../../assets/screenshots/graduation/eval-degree-audit-results.png)

**Review:**

- Check all required courses for the proposed credential.
- Confirm general education, major, and elective requirements as defined for that program.

> **Best Practice:** If multiple proposed credentials are being considered (for example, more than one stackable certificate), run separate EVALs for each PSPR scenario to avoid confusion.

---

### Step 5: Apply Standard Graduation Requirements to the "What-If" Program

Even in a "what-if" scenario, institutional graduation requirements still apply:

| Requirement | Standard |
|---|---|
| Minimum Program GPA | 2.0 (for the proposed program) |
| Minimum Cumulative GPA | 2.0 (all college-level work at CCCC) |
| Transfer Degrees (AA/AS/AE) | Every required course must be completed with a grade of "C" or better |
| Residency Requirement | At least 25% of the program's total credit hours must be earned at CCCC |
| Course Expirations | General five-year limits removed; technical courses may still have specific time limits (e.g., Health Information Technology) |

- Confirm that any time-limited technical courses fall within the allowed timeframe.

> **Talking Point:** "To award this additional credential, we must still confirm your GPA, and any program-specific course requirements."

---

### Step 6: Manage Course Substitutions for the "What-If" Program

If the "what-if" EVAL indicates missing requirements but you know an equivalent or substitute course has been approved:

**Actions:**

- Verify there is a Course Substitution Form on file with the Student Records and Registrar's Office.
- Do not force a course into the EVAL unless an approved substitution exists.
- If a substitution is approved, coordinate with:
  - **Registrar:** Jamie Childress
  - **Associate Registrar:** Haley Thomas
  - to have the course "forced" into the appropriate requirement area.

> **Talking Point:** "If your course is not showing correctly in this 'what-if' audit, we'll need to confirm whether a course substitution has already been approved before we adjust the evaluation."

---

### Step 7: Confirm Eligibility for the Additional/Stackable Credential

After applying the above standards:

- Review the EVAL results for the proposed program.
- Confirm all requirements (courses, GPA, residency, and any technical or transfer nuances) are fully met.

**If the student is not eligible:**

- Document missing requirements.
- Share next steps or remaining courses with the student.

**If the student is eligible for the new credential:**

- Note the exact program code and catalog used.
- Document the outcome in your advising notes or appropriate institutional system.

> **Talking Point:** "The 'what-if' evaluation shows that you have completed all requirements for this additional credential. Our next step is to ensure the program is officially added to your record and then move it forward for conferral."

---

### Step 8: Add the Program to the Student's SACP/SPRO Record

Before proceeding to conferral, the credential must be officially on the student's record:

- Work with the appropriate staff to add the program to SACP/SPRO if not already present.
- Confirm the correct program code, catalog, and award type are entered.

---

### Step 9: Link to Graduation Application and Upload to Advising Hub

Once the program is officially on record and eligibility is confirmed:

1. **Generate/Save the EVAL** — Save or print the completed EVAL as documentation of eligibility.
2. **Upload to Advising Hub:**
   - Locate the student's graduation application for the relevant credential.
   - Upload the EVAL document as part of the application record.

> **Requirement:** This upload is required for the application to move forward to final technical review and conferral.

## Key Colleague Screens for "What-If" EVALs

| Screen | Purpose in "What-If" Workflow |
|---|---|
| PSPR | Set up proposed "what-if" program and catalog |
| EVAL | Run the official degree audit for the proposed program |
| SACP | Verify existing active programs and awards |
| XNC2 | Identify Early College pathways as EC suffix is phased out |

---

## Quick Reference: "What-If" Scenarios

Use a "what-if" evaluation when:

- A student applies for a credential not listed on SACP/SPRO.
- You suspect the student has completed a stackable certificate or degree not already awarded.
